# Definition of Done

A task is complete when:

1. Acceptance criteria are satisfied.
2. Code compiles and passes tests.
3. No linting errors.
4. Security checks pass.
5. No console errors (frontend).
6. No obvious performance regressions.
7. Documentation updated if needed.
8. No dead code introduced.
9. No TODOs left unresolved.
10. Supervisor review completed.

If any of these fail → the task is not done.
