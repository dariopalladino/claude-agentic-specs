# UI Conventions

## Consistency

- Reuse shared components.
- Avoid multiple patterns for the same UI problem.
- Follow design system where available.

## States

Every async view must support:
- Loading state
- Error state
- Empty state
- Success state

## Forms

- Client-side validation.
- Clear error messages.
- Accessible labels.
- Disabled submit while processing.

## Accessibility

- Keyboard navigable.
- Proper semantic HTML.
- Sufficient color contrast.

## UX Principles

- Feedback for every action.
- No silent failures.
- Predictable navigation.
- No sudden layout shifts.

## Performance

- Lazy load heavy components.
- Avoid unnecessary re-renders.
